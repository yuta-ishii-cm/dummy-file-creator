# プロジェクトルール

## 実装後の動作確認

- コードを実装したら、必ず実際に動作確認を行うこと
- テストコードだけでなく、実際の使用シナリオで動作を確認する
- fgenツールの場合は、実装した機能を実際にコマンドラインから実行して確認すること
- 正常系だけでなく、異常系（無効な引数、存在しないファイルなど）も確認する
- ヘルプメッセージやエラーメッセージが分かりやすいか確認する
- 確認結果をユーザーに報告する際は、実行したコマンドと実行結果を具体的に示すこと

## プロジェクト構成

- このプロジェクトは、`fgen`というRust製CLIツールのリポジトリ
- 実際のRustコードは`fgen/`ディレクトリ配下にある
- ビルド・テストなどのcargoコマンドは`fgen/`ディレクトリで実行する必要がある

## CI/CD

- GitHub Actionsでテスト・フォーマット・Clippy・ビルドを自動実行
- `v*`形式のタグをpushすると、自動的にリリースが作成される
- リリース時は複数プラットフォーム（Linux/macOS/Windows）のバイナリを自動ビルド
